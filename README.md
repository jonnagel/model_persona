# Persona Inconsistency and OOD Behavioral Variance

A proof-of-concept studying whether fine-tuning on persona-inconsistent data
produces less predictable out-of-distribution (OOD) behavior compared to
persona-consistent training.

## Research Question

Does fine-tuning a language model on behaviorally inconsistent data (mixed
helpful + safety-refusal examples) cause higher-variance generalization on novel
prompts than fine-tuning on consistent data (helpful-only)?

## Method

- **Base model:** Qwen2.5-3B-Instruct (LoRA fine-tuning)
- **Training conditions:**
  - _Helpful-only_ — 750 consistently helpful examples
  - _Mixed/inconsistent_ — 375 helpful + 375 refusal-heavy examples, shuffled
  - _Baseline_ — unfine-tuned base model with a helpful system prompt
- **Data source:** Anthropic HH-RLHF
- **Probes:** 30 OOD prompts (ambiguous harm, identity, moral dilemmas, adversarial) + 10 in-distribution controls, 3 generations each
- **Metrics:** refusal rate, hedge density, response length, sentiment polarity, self-reference ratio

## Quick Start

### 1. Deploy to Lambda.ai

Follow the [Lambda deployment guide](https://docs.lambda.ai/public-cloud/) to set up an instance with GPU access.

Clone this repository onto the instance using either scp or git.

Note: A "gpu_1x_gh200" was used for the tests below, this selection was based on availability and
cost at the time. My total cost with tax was 2.74 USD.

```
### 2. Set up the instance

bash
ssh ubuntu@<lambda-ip>
cd ~/[project-folder]
bash setup_lambda.sh
```

This installs all Python dependencies, downloads the Qwen2.5-3B-Instruct model,
and pulls the Anthropic HH-RLHF dataset.

### 3. Run the full pipeline

```bash
bash run_experiment.sh
```

This runs all phases sequentially: data prep, fine-tuning (both conditions),
response generation, evaluation, and visualization. Outputs land in `eval/`.

### Running individual steps

```bash
python data_prep.py                        # Prepare training splits
python format_for_training.py              # Convert to ChatML
python train.py --condition helpful_only   # Fine-tune condition 1
python train.py --condition mixed_inconsistent  # Fine-tune condition 2
python quick_check.py                      # Verify training worked
python probes.py                           # Create probe sets
python generate_responses.py               # Generate all responses
python evaluate.py                         # Compute metrics
python visualize.py                        # Generate plots
```

## Results

### What the hypothesis got right

- **Mixed training produces higher behavioral inconsistency on OOD probes.** Per-probe refusal variance is 0.056 for mixed vs 0.0 for helpful — the mixed model sometimes refuses and sometimes doesn't for the same prompt. Per-probe sentiment variance is 1.23x higher.
- **Hedge density is the clearest continuous signal.** Mixed OOD variance is 70.7x its own ID variance, vs 21.6x for helpful. The mixed model hedges unpredictably.

### What's surprising

- **Helpful-only has zero refusals** across all 90 OOD responses. The helpful-only fine-tune completely suppressed refusal behavior.
- **Baseline actually has the highest overall variance** on several metrics (refusal, response length, self-reference). Fine-tuning itself — regardless of condition — reduces some variance dimensions.
- **The mixed model's elevated variance is concentrated in refusal and hedging**, not in response length or self-reference. The conflicting training signals specifically destabilize the refuse/comply decision boundary, not general fluency.

### Bottom line

The core hypothesis holds on 3 of 5 metrics: inconsistent training data produces measurably higher OOD behavioral variance in refusal, hedge density, and sentiment. But the effect is domain-specific (safety-adjacent behaviors) rather than uniformly elevated variance — a nuanced result that points toward targeted instability in the refuse/comply boundary as the primary mechanism.
